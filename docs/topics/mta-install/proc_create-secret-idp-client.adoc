= Create a secret for the IdP client

[role="_abstract"]
If you configured a secret in the Identity Provider (IdP) client configuration, enable the {ProductFullName} Hub to use the secret to connect to the Identity Provider (IdP) client.

.Prerequisites

* You installed {ProductShortName} {ProductVersion} in an OpenShift cluster.
* You have `admin` access to the cluster.
* You configured a secret in the IdP configuration for a secure connection.

.Procedure

. Create a secret such as `_<idp-client-secret>.yaml_`:
+
[source,yaml]
----
apiVersion: v1
kind: Secret
metadata:
  name: idp-client-secret
  namespace: <mta-namespace>
type: Opaque
stringData:
  clientSecret: "<secret>"
----
+
* `<mta-namespace>`:: The namespace where you deployed {ProductShortName}.
* `<secret>`:: The secret that you configured in the identity provider configuration.

. Apply the `_<idp-client-secret>.yaml_` file in the cluster:
+
[subs="+quotes"]
----
$ *oc apply -f `_<idp-client-secret>.yaml_`*
----

. Restart the Hub to load the secret:
+
[subs="+quotes"]
----
$ *oc rollout restart deployment/tackle-hub -n openshift-mta*
----

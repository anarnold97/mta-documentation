= Enabling agent mode

[role="_abstract"]
To verify code suggestions automatically with a local build dry run, enable agent mode in {mta-dl-plugin} through the Visual Studio Code command palette or the extension settings panel.

.Prerequisites

* You have installed the {mta-dl-plugin} extension in Visual Studio Code.
* Your project has a supported build file (`pom.xml`, `build.gradle`, or `package.json`) at the repository root.
* You have installed the corresponding build tool (Maven, Gradle, or npm) and added it to your system `PATH`.

.Procedure

. Enable agent mode in one of the following ways:

* By using the command palette:

.. Open the command palette by pressing Ctrl+Shift+P on Linux or Windows, or Cmd+Shift+P on macOS.
.. Type `Preferences: Open User Settings (JSON)` to open the `settings.json` file.
.. Set the `mta-vscode-extension.genai.agentMode` parameter to `true`.

* By using the extension settings panel:

.. Click *Extensions* > *{mta-dl-plugin}* > *Settings*.
.. Enable agent mode by selecting the *Agent Mode* checkbox.

. Restart the IDE language server when you are prompted to apply the change.

.Verification

. Open the Lightspeed output panel by clicking *View* > *Output* > *Red{nbsp}Hat Developer Lightspeed*.
. Request a code suggestion.
. Verify that the output panel displays entries similar to the following output:
+
[source,terminal]
----
[Agent Mode] Running build dry-run: mvn compile -q
[Agent Mode] Dry-run completed: SUCCESS (exit 0)
[Agent Mode] Applying verified suggestion to workspace
----

= Autocorrection on compilation failure

[role="_abstract"]
When a compilation step fails, the error is not immediately displayed. Instead, the {mta-dl-plugin} uses the compiler diagnostic output as additional context to generate a corrected suggestion from the {mta-dl-full} model.

This retry loop runs up to two times for minor errors, such as a single unresolved import or a mismatched method argument type.

If the suggestion cannot be corrected within the retry limit, agent mode displays the original compiler output alongside the suggestion and marks the code as unverified. You can then review and manually resolve the issue before you apply the change.

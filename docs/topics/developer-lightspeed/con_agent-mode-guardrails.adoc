= Output verification and agent mode guardrails

[role="_abstract"]
In agent mode, {mta-dl-plugin} includes an output verification mechanism that acts as a native compiler validation loop. This mechanism ensures that AI-generated code suggestions are syntactically valid and compatible with your classpath before they are written to disk.

The verification mechanism provides a programmatic guardrail against common large language model (LLM) risks, for example:

 * Hallucinated application programming interface (API) signatures
 * References to removed or deprecated methods
 * Syntactically broken output

== Agent mode code quality safeguards

When you run an analysis in agent mode, {mta-dl-plugin} passes each code suggestion through a local build dry run before applying it to your workspace. This process runs entirely on your developer workstation. No source code or build artifacts leave your local environment during verification.

The extension invokes the standard build toolchain that is already configured for your project directly in your local workspace:

* *Java projects*: Apache Maven (`mvn compile`) or Gradle (`gradle compileJava`)
* *JavaScript and TypeScript projects*: npm (`npm run build`) or Yarn (`yarn build`)

The dry run executes directly on the underlying file system of your local workspace without containerization or process isolation. If the dry run exits cleanly, {mta-dl-plugin} presents the verified suggestion for your approval before applying it to the workspace. If the dry run reports errors, agent mode attempts automatic correction.

== Agent mode validation checks

Agent mode performs the following verification checks on every code suggestion:

Syntax verification::
{mta-dl-plugin} parses the generated code against the grammar rules of the target language. Agent mode detects malformed tokens, unclosed blocks, or other parse-level errors at this stage before any file is written.

Method signature and reference checks::
The compiler resolves all newly introduced classes, method calls, field accesses, and variable references against your project's full dependency class path. This class path includes libraries that are declared in the `pom.xml`, `build.gradle`, or `package.json` files. This verification prevents hallucinated or deprecated API calls from entering the codebase.

Import and dependency consistency::
Agent mode confirms that every `import` statement that the suggestion adds corresponds to a type that is available on the class path. Agent mode flags missing or incorrect imports for correction.

== Autocorrection on compilation failure

When a compilation step fails, agent mode does not immediately surface the error to you. Instead, {mta-dl-plugin} feeds the compiler diagnostic output back into the {mta-dl-full} model as additional context and requests a corrected suggestion. This retry loop runs up to two times for minor errors, such as a single unresolved import or a mismatched method argument type.

If the suggestion cannot be corrected within the retry limit, agent mode surfaces the original compiler output alongside the suggestion and marks the code as unverified. You can then review and manually resolve the issue before you apply the change.
